# 角色认知域索引 IDX_COGNITION

> 版本戳：2026-07-22 覆盖 1–1000 章（单文件全量，不按卷拆分）
> 唯一「角色能知道什么 / 不能知道什么」检索入口。撰写新章前，先按本章 POV 角色与出场角色查本表，确定其**认知域边界**，杜绝写出「角色不可能知道的信息」（最易踩、也最易被自查漏检的逻辑坑）。
> 覆盖范围：与 `IDX_CHARACTER.md` / `IDX_SETTING.md` 同覆盖 1–1000 章（基于 v2 大纲全量重建，2026-07-21 建）。
> 本表是**权威数据源**，Stage1 必读、Stage3 逻辑矩阵 L1 对照本表核验（不靠 AI 凭记忆）。
> 与既有红线/禁复活/FB 正交：FB-002 穿越者绝对保密、FB-005 长生逐步深化，均在本表「不可知」列落地。

---

## §1 通用规则（全员适用）

- **R-cog-1 认知域约束**：任角色只写其**身份/年龄/修为/信息渠道/在场经历**决定的「可知」信息；不写其认知域外信息（如市井不知朝堂机枢、幼年不知政权运作）。
- **R-cog-2 限知 POV**：限知视角（尤其婴儿/儿童/单一配角 POV）不写该 POV 不可知的**他人内心**与**远处同时事件**；多 POV 须切换，不得让 A 无故知晓 B 独处时的心思或异地发生之事。
- **R-cog-3 不泄底**：不替读者/作者让角色「提前知道」未回收伏笔的答案（尤其 FB-002 穿越身份、FB-001 身世、FB-005 长生机制）；角色「猜到/隐约觉」可以，但「确知全貌」须待伏笔回收章。泄底 = FAIL。
- **R-cog-4 阅历一致**：角色内心活动/分析深度符合其阅历——婴儿无政权分析、市井无朝堂权谋、武者不突然懂火器工程原理（除非有合理信息源，如肖凡前世记忆限其内心独白）。
- **R-cog-5 信息差即张力**：配角「该知的要知道、不该知的留白」，制造悬念；反派有自身信息源但不全知，主角亦非全知（除特定阶段明示）。

## §4 写法约束速查（Stage2 落笔 / Stage3 核验）

1. **POV 限知**：每章先定 POV 角色 → 查 §2/§3 取其认知域 → 只写该域「可知」。
2. **主角分阶段**：先用章号算年龄 → 定位 S0–S4 → 取该阶段可知/不可知；不得让 S0/S1 主角写出 S3/S4 才该知的信息。
3. **多 POV 同步**：A 在甲地之所见，B 不得无故同现甲地知晓；切换 POV 须有章节/场景分隔。
4. **泄底判定**：叙事若写「X 知道了 FB-xxx 的真相/来历/全貌」而该 FB 在 `IDX_FORESHADOW.md` 仍标「未回收」→ FAIL（R-cog-3）。角色「似有所察/隐约不安」不在此限。
5. **受限外观**：婴儿/昏迷/背对等受限视角，不描写自身或不可见对象的外观细节，仅写触觉/听觉/嗅觉等可感知面（如 S0 玉佩只写「温润触感」）。
