# 小说创作流程体系（双对话 · 读者视角标尺）

> **定位**：本文件是《道士也玩火力覆盖》创作流程的**体系总纲**。
> 与既有文档关系：**作者侧四阶段细节以 `AI写作自动化流程.md` 为权威执行口径**；红线/纪律见 `写作规则.md`；本文件不重复二者已有内容，只补三块——
> ① 双对话架构（写作对话生成初版 / 审查对话审查+修复）与"审查对话"正式关卡；
> ② **「问题 → 流程改进」映射表**（把审查发现的硬伤反推成预防闸门，补的是"类"不是"个"）；
> ③ 审查对话 SOP 与验收标准（Definition of Done）。
>
> 本体系的形成直接源自审查实践（第一卷 81–100 店名断裂、117 章生成重复、FB-007 索引漂移、大纲年号漂移；第二卷 101–120 读者视角深读），是对已发生问题的闭环优化。

---

## 0. 总原则（铁律）

1. **双对话分工**
   - 写作对话 = **生成新章初版**（md 草稿），只做"从无到有"。
   - 审查对话（本对话）= **审查 + 修复既有内容**：机械校验、读者视角深读、一致性/红线/索引漂移审计，发现问题直接落盘修改。
2. **磁盘 = 唯一事实源**：两对话通过 `md / txt / 治理索引` 交接，不靠对话记忆。
3. **读者视角 = 审查唯一标尺**：机械校验 / 索引章号 / 大纲笔误属**后台自检**，不计入读者审查报告；只有当它们**引发读者出戏**时才升格为审查项。
3.1 **读者审查全维度无死角**：读者能感知的一切点都纳入审查——**爽感 / 节奏 / 人物 / 世界 / 文字 / 伏笔** 6 大类共 24 项（见附录 A）。不得只查叙事手法/红线而漏掉节奏断崖、人设 OOC、文字流畅、伏笔回收、时空可信等读者真在乎的维度；任一维度命中 FAIL 即记为审查发现。
4. **Ardot 视觉管线不介入**：纯文本项目，ardot-design-core / router 不触发画布、不做视觉生成。
5. **发现问题直接修改（审查对话权限）**：
   - **硬伤**（事实矛盾 / 生成伪影 / 红线触碰 / 索引漂移）→ 直接落盘修；
   - **创作取向建议**（章末钩子增强 / 修炼章节奏 / 反派密度）→ 给写作对话执行，不擅自改写正文基调。另含：高潮金句可适度（非鸡汤、合人物声口，见写作规则 §五）；预期读者争论点预测（本地版，替代番茄评论区诱导，供写作对话参考情绪/争议设计）。

## 1. 双对话架构与双闭环

> **双闭环总览**（读者审查已融合进写作对话，形成完美闭环）：
> - **第一道关 · 写作对话内闭环**：阶段一 准备 → 阶段二 写作 → 阶段三 自检 + **沉浸式读者审查** → 发现问题**就地修改** → **问题反补流程** → 阶段四 提交「已自审稿」。大部分问题在源头消灭。
> - **第二道关 · 审查对话外闭环**（本对话）：对「已自审稿」做独立复核，专抓写作对话盲点（生成伪影 / 隐性 OOC / 读者自查易放过的破墙）→ 直接修复 → 反补流程。
> 两闭环共用 §2「问题→改进表」同一套预防闸门，形成「写作 → 审查 → 修改 → 反补 → 再写作」的闭环。

```
[用户触发：指定章号]
        │
        ▼
┌─────────────────────────────────────────────┐
│ 写作对话：阶段一 准备 → 阶段二 写作 → 阶段三 自检 │
│           → 阶段四 提交(用户确认 md 初稿)        │
└─────────────────────────────────────────────┘
        │ 用户确认 md 初稿（不自动出 txt）
        ▼
┌─────────────────────────────────────────────┐
│ 审查对话（本对话）：阶段五 审查 + 修复           │
│  · 量化扫描(字数/钩子/破墙词/对话比)            │
│  · 机械校验 logic_check C1–C22                 │
│  · 逐章读者视角深读                            │
│  · 索引交叉比对(FB/伏笔/角色/设定 vs 正文)      │
│  · 发现问题 → 硬伤直接修 / 取向给建议          │
│  → 输出《读者视角审查报告》                     │
└─────────────────────────────────────────────┘
        │ 验收(DoD)达成
        ▼
┌─────────────────────────────────────────────┐
│ 落盘：写作对话按"确认后"生成 txt + 回填索引      │
│      （或审查对话修完即标记"可出 txt"）          │
└─────────────────────────────────────────────┘
```

> **说明**：阶段一至四执行细节见 `AI写作自动化流程.md`（其阶段三已升级为「写作自检 + 沉浸式读者审查」内闭环、含「问题反补闸门」）；阶段五（审查对话·第二道关）为本体系新增关卡，详见 §3。写作对话交出的 md 已是「经读者审查自修后的稿」，审查对话只做独立复核与盲点深挖。

## 3. 审查对话 SOP（本对话）

**输入**：写作对话产出的 md 初稿 + 磁盘索引/大纲（唯一事实源）。

**步骤**：
1. **量化扫描**（脚本/命令）：字数、章末钩子命中率、破墙词（R19 现代词，经营理念词按 `IDX_PERSONA.md` §1 锁词表放行不计）、对话比、钩子类型相邻 3 章不重复（见 §3 钩子类型库）。
2. **机械校验**：`python logic_check.py <章md>` → C1–C22 全过。
3. **逐章读者视角深读（全维度 24 项，见附录 A）**：爽感（爽点密度/质量·情绪曲线·悬念信息差·穿越反差）、节奏（章内节奏·章末钩子·张弛分配·信息密度·战斗呈现）、人物（人设OOC·动机合理·对话自然·配反质量·成长可见）、世界（破墙出戏·场景具体·世界观一致·称谓关系）、文字（流畅·时空可信·因果逻辑）、伏笔（伏笔管理·角色去向·长生机制）。逐项判 PASS/FAIL + 证据，命中即写入《读者视角审查报告》。
4. **索引交叉比对**：FB 码 / 伏笔 / 角色 / 设定 与正文一致，无悬空码、无章号漂移。
5. **处置**：
   - 硬伤 → **直接落盘修改**（本对话权限）；
   - 创作取向 → 给写作对话执行（不改正文基调）。
   - **修复后独立复核（防假通过闸门）**：凡声称"已修复同构/已差异化/已去重/已打破模板"的，必须：①主对话**独立读实际 N 章文本**确认（不依赖改写方 Summary 自验）；②在报告**附 Before/After 片段或逐句 diff 作证据**，纯断言（如"各章句式不同"）＝未修复；③跑**跨章反例扫描**：对模板特征词（`坊后空场|肩臂松了松|金身已成|身基与.*基一般|炼体十八式|春寒映窗|阖眸|指尖轻叩`）跨章 grep，命中 >2 章即打回重做；C22 扩至跨章相邻 ≥20 字重复块扫描。
   - **修复后强制字数复验（闸门）**：凡本对话落盘修改的章，修改完毕**必须实跑纯汉字字数验证**，确认单章纯汉字 **≥2500**（项目下限，目标约 3000）；**不达标则继续补写至达标，严禁凭估计放行**。脚本口径与 `logic_check` 字数阈值一致——`re.findall(r'[一-鿿]', t)` 计纯汉字（去换行后总字符作辅助参考）。此法直接源于 153 章压缩过度（~2501→1033）的教训：修复不得引入"半章"。
6. **输出**：`审读_NNN-NNN读者视角.md`（纯读者视角；后台自检不计入），报告**须登记本次修复的字数复验结果**（修复前/后纯汉字数 + 是否达标），未达标项不得标记"可出 txt"。

**直接修改权限清单**：店名/专名统一、年号/年龄校正、索引章号同步、生成重复/硬伤修复、R19 非例外现代词清除（经营理念现代词按 `IDX_PERSONA.md` §1 锁词表放行，不在此列）。

## 4. 写作对话预防闸门（新增，已嵌入 `AI写作自动化流程.md`）

- **阶段三 沉浸式读者审查（内闭环核心）**：从"清单逐条核对"升级为"以首次读者身份逐章追更模拟"，发现问题**就地修改**后重跑至 PASS；确保交出"已自审稿"而非裸初稿（详见 `AI写作自动化流程.md` 阶段三 / `novel-chapter-write/SKILL.md`）。
- **阶段三 问题反补闸门（闭环优化·强制）**：发现的**新问题类型**必须反补——可机检 → 提 `logic_check.py` C 码需求；预防类 → 加 `novel-chapter-write/SKILL.md` 防复发清单；流程缺漏 → 改 `AI写作自动化流程.md` / `写作规则.md`；红线例外 → 登 `IDX_PERSONA.md` 锁词；并沉淀 §2「问题→改进表」。

- **阶段二 防复发自检清单** 增补：
  - ⑪ **专名 canonical 一致性**：商号/人名/地名写前取大纲/索引 canonical 名，写后 `grep` 全卷比对，禁止旧名/别名回潮。
  - ⑫ **章内句级重复**：`logic_check` C22 须无重复句。
- **阶段三 机械校验**：C 码范围扩展到 **C1–C22**（含 C22 句级重复、C23 专名回归）。
- **阶段三 自检清单** 增补维度 14：**专名 canonical 一致性**（跨章比对）。
- **阶段三 维度 #9 章末钩子**：强化定义 + 验收（末句须为截断悬念/未知/危机）。
- **阶段四 索引同步**：内容变更即 `diff` 索引（grep 正文 FB 码出现章 vs 索引登记章），非仅手动回填。
- **大纲撰写（`写作规则.md` §十六）**：自查清单增第 8 项「元信息自洽」（年号/年龄与正文一致、章名与文件名一致）。

## 5. 验收标准（Definition of Done）

一章视为"可出 txt"当且仅当：

- [ ] `logic_check`：C1–C6 / C8 / C11 / C18 / C19 / C20 **无 FAIL**；C7/C9/C10/C12–C17/C21/C22/C23 的 WARN 已显式回应；
- [ ] **专名回归**：`grep -rnE "肖记|遥遥领先" 卷目录/` **零命中**（canonical＝好物商城等现行名）；
- [ ] **年号/年龄自洽**，章名与文件名一致；
- [ ] **R19 红线**：零非例外现代词（经营理念现代词按 `IDX_PERSONA` §1 锁词表放行、仅限肖凡；现代实物名词/网络梗仍禁）；
- [ ] **索引与正文一致**（FB 码/伏笔/角色/设定无漂移）；
- [ ] **读者视角无硬伤**：无重复/无破墙/章末钩子到位/节奏不连章断崖；
- [ ] **字数复验（修复后强制）**：本对话落盘修改的章，**实跑纯汉字计数 ≥2500**（单章下限，目标约 3000），不达标不准出 txt；审查报告须登记复验结果（修复前/后纯汉字数 + 是否达标）。
- [ ] **反模板复验（防假通过强制）**：凡涉及"打破模板/去重/差异化"的修复，须在报告中**附 Before/After 证据**，且**跨章反例扫描通过**（模板特征词 `坊后空场|肩臂松了松|金身已成|身基与.*基一般|炼体十八式|春寒映窗|阖眸|指尖轻叩` 跨章命中 ≤2 章）；纯断言无证据者不准出 txt。

## 6. 反馈闭环（双闭环共灌）

> **双闭环共灌**：写作对话内闭环的反补（阶段三 #7 / SKILL 4.5）与审查对话外闭环的反补（§3 步骤 6 → §2 映射）汇入**同一张「问题→改进表」**——写作对话在自检/读者审查中发现的新类型问题，当场反补，下一章即生效；审查对话发现的盲点问题，反补后同样下一章生效。两股反补给同一套预防闸门"充电"，闸门越用越密，同类问题不再复发。

```
[写作对话内闭环] 自检/读者审查发现 → 反补(阶段三#7) ┐
                                                      ├─→ 映射「问题→改进表」→ 无对应闸门则新增(补类)
[审查对话外闭环] 独立复核发现盲点 → 反补(§3步骤6) ┘        │
                                                      ▼
                                          沉淀至 写作规则.md / 本体系(共性) / IDX_*(索引漂移)
                                                      │
                                                      ▼
                                          写作对话下一章自动规避(预防闸门生效)
```

## 附录 A · 全维度读者审查清单（6 类 24 项）

**定位**：读者视角审查的**唯一权威定义**。写作对话（阶段三·沉浸式读者审查）与审查对话（§3 逐章深读）均以此 24 项为刻度，覆盖**读者能感知的一切点**——任一项"读者不应看到"命中即 FAIL。

**全维度无死角**：不得以"后台自检/红线/手法"替代读者审查；节奏断崖、人设 OOC、文字流畅、伏笔回收、时空可信等读者真在乎的维度，与破墙/爽点同等权重。

| # | 类别 | 审查项 | 读者**应看到**（PASS） | 读者**不应看到**（FAIL） |
|---|------|--------|------------------------|--------------------------|
| A1 | 爽感 | 爽点释放 | "众人轻视→被打脸"过瘾瞬间、认知颠覆、升级/装逼可感 | 爽点被提前泄气、反派降智到不可信、全章无爽点 |
| A2 | 爽感 | 情绪曲线 | 整章有起伏（紧张→舒缓→高潮），情绪有推动 | 全程平/全程丧/无波动，读来麻木 |
| A3 | 爽感 | 悬念与信息差 | 伏笔铺陈后读者自己拼出答案；权谋信息不对等 | 直白喂结论、信息降维；底牌同时亮明无信息差 |
| A4 | 爽感 | 穿越反差（本书特定） | 读者独享"现代思维 vs 古代"反差爽感 | 任何角色识破穿越者、或泄底 FB-002 |
