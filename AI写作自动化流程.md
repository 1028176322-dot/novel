# AI 自动化章节写作流程（四阶段闭环）

> 适用范围：《道士也玩火力覆盖》各卷正文撰写（当前重点：第六卷·乾坤 851–1000 章）。
> 设计目标：把"写作规则.md + v2 大纲 + 五张治理索引（伏笔/角色/设定/认知/人设）"固化为**可机检的四阶段流水线**，每阶段有明确输入/输出、进入门槛（gate）与衔接触发逻辑，杜绝"没读规则就写""写完不自检""跳过确认直接出 txt"。
> 本流程与 `写作规则.md` §三（写作流程）、§四（写作纪律）完全一致，是对既有规则的**强制编排**，不新增任何创作约束。

---

## 0. 总览与流转

```
[用户触发：指定要写的章号，如 851 / 851-865]
        │
        ▼
┌─────────────────────────────────────────────┐
│ 阶段一 · 写作前准备  (准备态)                  │
│  载入：规则 + 本卷卷首 + 本段大纲 + 前序章节    │
│       + 四索引相关条目 + 禁复活/红线           │
│  GATE：必读清单 100% 已加载且无缺失 → 否则中止  │
└─────────────────────────────────────────────┘
        │ 通过
        ▼
┌─────────────────────────────────────────────┐
│ 阶段二 · 章节正文写作  (作者态)                │
│  以作者身份产出草稿 md，严格贴合 v2 大纲节拍   │
│  GATE：正文非空 + 字数达标 + 含章末钩子 → 否则 │
│        补写后进入阶段三                        │
└─────────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────────┐
│ 阶段三 · 写作自检  (审查态)                    │
│  逐条对照写作规则自检，生成自查报告            │
│  GATE：全部条目 PASS → 进阶段四                │
│        任一条 FAIL → 回阶段二修订（带 FAIL 项）│
│        修订循环上限 N=3，超限升级人工仲裁       │
└─────────────────────────────────────────────┘
        │ 全部 PASS
        ▼
┌─────────────────────────────────────────────┐
│ 阶段四 · 人工审核提交  (待审态)                │
│  提交：草稿 md + 自查报告                      │
│  等待用户反馈 → 打回则按意见回阶段二/三         │
│  用户通过 → 生成 txt（确认后才出）→ 更新索引   │
└─────────────────────────────────────────────┘
```

**核心铁律（来自写作规则.md，不可被流程绕过）：**
- 红线 R1–R21 全程严守；禁复活名单角色绝不安排出场/复活。
- 章节正文（第*.md）写完**必须**在回复中贴出逐条自查结果，否则视为未自查、用户可直接打回。
- txt 文件**仅在用户确认后**生成（既有约定：md→确认→txt），AI 不得自主出 txt。
- 扩写/修改章节同样走完整四阶段，不因"只是补充"而跳过自检。

## 阶段一 · 写作前准备

**角色**：加载器 / 合规守门员（非作者）。
**目标**：确保动笔前所有约束素材已就位，缺任何一项都不准进入写作。

### 输入
- 用户指定的目标章号（单章或连续段，如 `851` 或 `851-865`）。
- 由章号可推导的元信息：所属卷（第六卷·乾坤）、所属大纲段文件、章节在段内序号。

### 必读文件清单（全部加载才算"准备完成"）

| # | 文件 | 路径（项目相对） | 用途 | "已加载"判定 |
|---|------|------------------|------|--------------|
| 1 | 写作规则 | `写作规则.md` | 全局文风/红线/禁复活/格式/自查标准 | 文件存在且 §三·§四 关键小节已读取 |
| 2 | 项目总规则 | `PROJECT_RULES.md` | 项目级补充约束 | 文件存在并读取 |
| 3 | 卷首说明 | `第六卷_乾坤/大纲/大纲_v2_卷首说明.md` | 五幕骨架、R1–R21、FB-110~125 主伏笔、Option B 年表铁锚 | 文件存在且"五幕/红线/年表"三节已读 |
| 4 | 本段大纲 | `第六卷_乾坤/大纲/大纲_v2_第{起}-{止}章.md` | 本章逐章节拍、爽点标记、红线引用、撰写备注 | 目标章所在段文件已读，定位到具体章条目 |
| 5 | 前序章节 | `第六卷_乾坤/第{前序}章_*.md`（已写部分） | 连续性：人物状态/地点/未回收伏笔 | 目标章之前已存在的最新 1–3 章已读 |

| 12 | 风格融合手册 | `参考资料/风格融合手册.md` | 外部作品风格提取的**唯一累积源**；写章前取笔法增强（慢热日常/史笔双层/碎片蒙太奇/神态动作标签/冷面打趣/留白悬念等），应用时守红线 | 文件已读；本章拟用到的笔法点已对照手册「二、本作硬约束」过红线 |

### 执行步骤
1. 解析章号 → 定位卷与大纲段文件（清单 3、4）。
2. 读取清单 1–4（固定基础约束）。
3. 读取清单 5（前序连续性）。
4. 从大纲段文件提取本章 FB 号/角色/设定关键词 → 读取清单 6–8 对应条目。
5. 校验清单 9：本卷大纲必须为 `current`（非 blueprint），否则中止并报错。
6. 输出《准备完成清单》：12 项逐条 ✅/❌。

### 输出
- 《准备完成清单》（11 项加载状态表）。
- 一组"已加载上下文"：规则要点、本章大纲节拍、相关 FB/角色/设定快照、连续性备忘录。

### 进入阶段二的门槛（GATE-1）
- **全部 12 项 ✅**，且本卷大纲状态=current。
- 任一项 ❌（文件缺失/段落未读/大纲非 current）→ **中止并报告缺项**，不进入写作。

## 阶段二 · 章节正文写作

**角色**：作者（第一人称叙事，贴合肖凡视角与男频爽文文风）。
**目标**：产出一章合规草稿 md，严格贴合已加载的 v2 大纲节拍。

### 输入
- 阶段一全部"已加载上下文"（规则 + 大纲 + 索引快照 + 连续性备忘）。
- 若从阶段三回退：附《FAIL 修订项清单》。

### 认知边界预检（动笔前强制 · 防 L1 类逻辑问题）

落笔**之前**必须按 `IDX_COGNITION.md` 明确本章**所有出场角色**的认知域边界——认知不是主角专属，而是**全员分层**：每个角色能知道什么，由其身份/年龄/修为/信息渠道/在场经历决定。**这是最易踩、也最易被自查漏检的逻辑坑**。

- **主角肖凡（按年龄阶段分层）**：先用章号算年龄（公元−1370）定位 S0–S4 阶段，取该阶段「可知/不可知」——S0 弃婴只写自身触觉/听觉+前世记忆（内心），不写世界全貌、不描写自身不可见物外观；S1 童年不懂政权机枢；S2 游历不写守源/九枚全貌；S3 中年掌局仍有未知；S4 老年近乎全知但**不替读者提前知余七枚去向（留白）**。
- **其他角色**：各按 `IDX_COGNITION.md` §3 取其认知域——如无为子不知肖凡穿越（FB-002）、萧俊至死不知自己被寒髓引毒（FB-115）、市井不知朝堂大局、反派有自身信息源但不全知。
- **通用红线**：限知 POV 不写他人内心/远处同时事件（R-cog-2）；不替读者让角色提前确知未回收伏笔答案（R-cog-3，尤其 FB-002/FB-001/FB-005）；内心活动深度符合阅历（R-cog-4）。
- 输出《认知边界备忘》（逐角色）：
  ```
  POV=___（年龄/阶段：___，查 IDX_COGNITION §2/§3）
    ├─ 可知=___（依该角色认知域）
    ├─ 不可知=___（含未回收伏笔答案/域外信息/他人内心）
    ├─ 其他出场角色认知边界是否越界：___（如 A 不该知 X）
    ├─ 空间锚点=___｜可达边界=___（R11，防越界感知/瞬移/跨场景无故现身）
  ```
  随草稿一并提交自检对照。

### 人设行为边界预检（动笔前强制 · 防 L4 类逻辑问题）

落笔**之前**必须按 `IDX_PERSONA.md` 明确本章**所有出场角色**的性格—行为边界——这是"角色像不像、该不该这么干"的权威判据，最易被"作者为了方便强行安排角色行为"踩坑。

- **主角肖凡（现代灵魂）**：先取 `IDX_PERSONA.md` §1——现代思维**仅限内心独白与反差爽点**，台词/旁白**零现代实物名词、零网络梗**（R19；R19 例外：经营理念现代词仅限肖凡，见 `IDX_PERSONA.md` §1 锁词表）；绝不泄穿越（FB-002）；维持藏拙（R17）；言行须符合其年龄阶段性格弧（S0 懵懂内心吐槽 → S1 内敛观察 → S2 务实谋断 → S3 沉稳布局 → S4 通透）；对外半文半白适配古代场合、简洁留余地。
- **其他主要角色**：各取 `IDX_PERSONA.md` §2 其档案——性格标签（主标签必须成立）、说话风格（声口特征）、行为决策模式、雷区禁忌（如无为子不涉权谋、萧俊至死不知 FB-115 寒髓引毒、苏墨凝不早于第082章首现且不情绪化硬刚）。
- **反派**：取 §3 通则——**不降智、有自身逻辑**（夺嫡/自保/部族存亡），决策在自身认知域内合理，不送爽点；禁复活角色零复活（萧沛仅余党裴琰承线）。
- **新出场角色**：须带"独特性格标签 + 声口特征"（§5 多元化图谱提示可补"幽默/跳脱"等未被充分覆盖维度），禁止无性格工具人。
- 输出《人设行为备忘》（逐角色）：
  ```
  角色=___（查 IDX_PERSONA §2/§1）
    ├─ 性格标签=___（主标签须成立）
    ├─ 说话风格=___（声口特征）
    ├─ 行为是否由性格+处境驱动（非作者强排）：___
    ├─ 是否触雷区（如萧俊知毒/无为子权谋/反派降智）：___
    ├─ 主角专项：现代痕迹仅内心？零现代词破氛围？未泄穿越？随年龄弧？：___
  ```
  随草稿一并提交自检对照（Stage3 L4 逐条回应）。

### 执行步骤
1. 以"作者"身份动笔，严格按大纲段文件本章条目中的**情节节拍分解**逐拍展开。
2. 文风对齐 `写作规则.md` §五（语言风格、禁止作者视角、叙事视角统一、因果链、场景化）。
3. 红线对齐：涉及科技/火器/越阶战须对照 R1–R21 与 FB 例外约束（如 R4/R5、狙击枪 FB-065 唯一例外）。
4. 禁复活：出场人物仅限角色索引中"非禁复活"且本章大纲列出者；新角色按索引占位名（如"新国师"）先留白。
5. 爽点与钩子：每章 ≥1 小爽点（呼应大纲"爽点标记"）；章末必须留悬念/危机/未知（下章钩子）。
6. 字数：单章 **不含标点的纯汉字 2500–3000**（不以含标点的字符数计），不低于节拍数×450 的纯汉字下限。
7. 若承接阶段三回退：逐条处理 FAIL 项，不改写其他已合规部分。
8. **写完扫描（防备注漏入正文）**：落笔后通扫全章，凡形如 `（节拍…）` / `（R\d+：…）` / `（章末钩子…）` / `（悬念钩子…）` 等"作者写作备注/手法标注/红线标记/钩子说明"括号行，一律删除——此类文字只存在于大纲与《手法调度表》，**绝不进正文**（本条为读者视角审查"正文纯净度"项的源头预防）。

### 输出
- 章节草稿：`第六卷_乾坤/第{章号}_《章名》.md`（章名取自大纲段文件本章条目）。
- 正文为**纯净叙事**：不嵌「本章概要/爽点/情绪曲线/章末钩子」元信息块——此类结构信息已完整存在于 v2 大纲模板（本章核心任务 / 冲突 / 情节流程 / 情绪变化 / 收获代价 / 伏笔钩子），无需在正文重复。章末钩子 = 叙事最后一句自然留出的悬念，不另标块。

### 进入阶段三的门槛（GATE-2）
- 正文非空、字数达标、**含强定义章末钩子**、且**无写作备注混入正文** → 进入阶段三。
  - **章末钩子强定义**（2026-07-23 强化，防 101–120 钩子缺失类）：末句须为「截断悬念 / 未知 / 危机」三类之一，不能是平铺总结或情绪收束；量化扫描钩子命中率（见 `docs/流程体系_小说创作.md` §3），缺失则补写后进入阶段三。
- 任一项不达标 → 补写后直接进入阶段三（不回退阶段一，上下文仍有效）。

## 阶段二 · 附 · 叙述手法调度规范（Narrative Mode Dispatch）

> 质量不稳、手法错配的根因：AI 默认用同一种腔调写所有场景。本规范把"写什么"（大纲节拍）与"怎么写"（叙述手法）解耦——**写作前先判定每段节拍的手法类型，动笔前先列《手法调度表》，自检时核查手法适配**。与既有红线/禁复活/FB/读者视角审查正交，不新增创作约束。

### 剧情类型 → 叙述手法映射库

| # | 剧情类型 | 叙述手法 | 节奏 | 信息/视角 | ✅ 做 | ❌ 不做 |
|---|---------|---------|------|----------|------|--------|
| 1 | 战斗/打脸爽点 | 结果速写法 | 快·短句快切 | 读者全知结果，对方震惊特写 | 写"他赢了"的果 + 对方表情/崩塌 | 招式拆解注水、慢镜头拖戏 |
| 2 | 权谋/暗线 | 信息差叙事 | 中慢 | 读者＞角色，留白 | 密信、侧写、未说破的暗示 | 所有底牌同时亮明、泄气 |
| 3 | 揭秘/身世 | 渐进揭示 | 由慢到快 | 读者随角色拼图 | 一层层剥、用角色感受带动 | 一次全盘托出、作者旁白解说 |
| 4 | 日常/过渡 | 白描推进 | 稳 | 客观 + 细节 | 用细节推关系/埋伏笔 | 概括式流水账、纯过场 |
| 5 | 情感/羁绊 | 内聚焦留白 | 慢 | 角色内视 | 动作/物象代情绪（show） | 直抒"他很伤心"（tell） |
| 6 | 高潮/大场面 | 多线并切+情绪弧 | 由压到爆 | 多 POV 快切 | 先压后炸、群像反应 | 单视角平铺、无情绪曲线 |
| 7 | 设定 exposition | 场景化露出 | 自然 | 借角色冲突/对话 | 设定从对抗/疑惑中自然掉 | 作者跳出教科书解说 |
| 8 | 章末钩子 | 截断悬念 | — | 信息刚露即停 | 在转折/揭秘临界点断 | 把答案先说了再断 |

### 单章节奏配速七步（模板）

钩(开头 1–2 段抓眼) → 起(铺垫/状态) → 承(推进/信息) → 压(张力积聚/对立) → 转(转折/意外) → 爽(打脸/揭晓/高潮) → 钩末(章末悬念)。
- 单章 **不含标点的纯汉字 2500–3000**，钩与钩末各约占 5–10%，中段按节拍分配；连续章不得同构（见阶段三 #12）。

### 文风基线 + 硬约束速查卡（Stage2 写作时随身对照）

- **文风基线**：半文半白，白话叙事 + 适度文言对话；现代词仅限肖凡内心独白，不得入叙事旁白或他人台词；show don't tell、场景化、因果链完整；禁用作者旁白解说、现代网络梗、破氛围的现代实物名词。
- **风格融合（consult `参考资料/风格融合手册.md`）**：写章前取已沉淀笔法增强——限知+史笔双层（章首/转场「永熙X年，大晟……」定调）、慢热日常密度、碎片记忆蒙太奇、神态动作标签密、冷面打趣+潜台词、冷峻+温情闪现、`……`留白悬念。所有笔法须过手册「二、本作硬约束」红线（R1–R3/R14/R17/R19、年号≤37、空间锚点清虚观/盛京、FB-002 保密）。
- **硬约束速查**：红线 R1–R21（重点 R1 科技碾压 / R2 无原生火药 / R4 火器对宗师偶发致命 / R5 研发时间线 / R11 空间锚点 / R17 布局者 / R19 工业党 / R20 穿越者 / R21 自查）；禁复活名单（萧沛禁复活仅余党，其他禁复活角色零出场）；FB-002 穿越者绝对保密；FB-005 长生逐步深化不剧透；FB-110+ 不重定义、新 FB 走专属号段；年号永熙≤1404 / 昭明 1405–1456 / 景和 1457+（不写永熙超 37 年）。

## 阶段三 · 写作自检 + 沉浸式读者审查（写作对话内闭环）

**角色**：审查员（对照规则逐条核验，不与作者身份混淆）+ **首次阅读的读者**（跳出作者视角，以读者身份逐章追更）。
**目标**：全面自检 + **以读者身份沉浸式审查**，发现问题**就地修改**，并将新发现问题类型**反补流程**；不合格则回退修订。这是写作对话的**第一道关**——把大部分问题在源头消灭，而非推给审查对话；审查对话只作为独立第二双眼睛专抓盲点。

### 输入
- 阶段二产出的草稿 md。
- 阶段一加载的全部规则与索引上下文。

### 读者视角审查（强制块 · 该看到 / 不该看到）

**目的**：跳出"作者/审查员"视角，以**首次阅读的读者**身份审视——读者读这章时，应当获得什么体验、不应当被什么破坏体验。任一项"读者不该看到"命中即 FAIL。

> **全维度无死角原则**：上表 6 类 24 项覆盖**读者能感知的一切点**——爽感体验、节奏结构、人物可信、世界沉浸、文字可读、伏笔回收。任一项"读者不该看到"命中即 FAIL。权威定义与分类框架见 `docs/流程体系_小说创作.md` 附录 A。

> 审查方法：把本章当"没写过它"来读一遍，逐行问——**这句话是读者想看到的，还是只有作者/AI 才知道的？** 凡属后者且非角色应有认知，判 FAIL。
> **发现问题就地修改（内闭环核心）**：读者视角审查（全维度 24 项）任一项命中 FAIL → **立即修改对应正文**，修改后重跑该维度至 PASS；不另起草稿、不依赖审查对话。这是写作对话"审查→修复"内闭环，确保交出的 md 已是"经读者审查自修后的稿"。

> 机制说明：L2/L6 与"正文纯净度"由 `logic_check.py` 机械校验（见阶段三执行步骤 #0）；**空间/位置(L9) 由 logic_check.py C7 提供「空间锚点缺失 / 互斥地点冲突」WARN 辅助**，AI 须据此在 L9 逐条显式回应。**角色认知边界(L1) 由 `IDX_COGNITION.md` 提供全员分层权威数据**，AI 须按本章 POV 角色与年龄阶段逐条回应（主角 S0–S4、其他角色 §3 认知域）。**角色性格与行为逻辑(L4) 由 `IDX_PERSONA.md` 提供全员性格档案权威数据**，AI 须按本章每个出场角色逐条回应其 §2/§1 档案与 §4 OOC 核查清单（主角现代灵魂边界、反派不降智）。L1/L3/L4/L5/L7/L8/L9 为语义核查，AI 须逐条给 PASS/FAIL + 证据。每次用户反馈的逻辑 bug，先映射到上表某一类——**补的是"类"不是"个"**，避免同类问题反复出现。

### 执行步骤
0. **机械校验（强制先于一切 AI 核查）**：运行 `logic_check.py <本章md路径>`（脚本路径见附录 A），输出 **C1–C22**（含 C22 章内句级重复、C23 专名回归，见 `docs/流程体系_小说创作.md`）。其中 **C1–C6、C8、C11、C18、C19、C20 任一项 FAIL → 直接回阶段二修订，不进入后续 AI 核查**（此六项为硬 FAIL）；C7、C9、C10、C12、C13、C14、C15、C16、C17、C21、C22 为 WARN 级，须在 L9 / 读者视角审查 / 字数·章名核查 逐条显式回应（机械项不靠 AI 记忆，最可靠）。
1. 逐条运行上表（13 维）核验，每条给 PASS / FAIL + 证据（引用正文行或规则条款）。
2. 运行"读者视角审查"块（全维度 24 项），逐行判定应见/禁见，给 PASS / FAIL。
2.5 **发现问题就地修改（内闭环）**：上一步或 13 维 / 逻辑矩阵任一项 FAIL → 立即修改对应正文（不另起草稿），修改后重跑该维度至 PASS；这是写作对话"审查→修复"环节，不依赖审查对话。
3. 运行"逻辑一致性防御矩阵"（L1–L9），逐类对照数据源给 PASS / FAIL + 证据（L9 空间须逐条回应 C7 的 WARN）。
4. 汇总为《自查报告》（含 13 维 + 读者视角 + 逻辑矩阵 + 机械校验结果 + 总体结论）。
5. 若全 PASS → 报告结论"✅ 通过，可提交"。
6. 若有 FAIL → 报告列《FAIL 修订项清单》（条目号 + 问题 + 修改方向），不带草稿进入写作修订。
7. **问题反补闸门（闭环优化·强制）**：本次审查发现的**新问题类型**（非一次性笔误），必须反补——① 可机检 → 提 `logic_check.py` C 码需求（或扩 C22 类句级检查）；② 预防类 → 加 `novel-chapter-write/SKILL.md` 防复发清单条目（⑬⑭…）；③ 流程缺漏 → 改本流程文件 / `写作规则.md`；④ 红线例外 → 登 `IDX_PERSONA.md` 锁词。一次性笔误不沉淀。凡反补，同步 `docs/流程体系_小说创作.md` §2「问题→改进表」新增一行（补"类"不是"个"）。

### 输出
- 《自查报告》（内嵌于回复，强制贴出）。
- 若 FAIL：附《FAIL 修订项清单》交回阶段二。

### 衔接触发逻辑（GATE-3）
- **14 维 + 读者视角审查(全维度 24 项) + 逻辑一致性防御矩阵(L1–L9) 全部 PASS，且 `logic_check.py` 机械校验（C1–C6、C8、C11、C18、C19、C20 无 FAIL；C7/C9/C10/C12/C13/C14/C15/C16/C17/C21/C22 的 WARN 已在 L9 / 读者视角审查 / 字数·章名核查 逐条显式回应），且问题反补闸门(步骤#7)已执行** → 触发阶段四（提交）。
- **任一 FAIL**（含读者视角审查任一项命中、逻辑矩阵任一类 FAIL、或机械校验任一 C 项 FAIL）→ 回退阶段二，携带 FAIL 项；修订后重新进入阶段三（不自退回阶段一）。
- 修订循环上限 **N=3**：连续 3 次仍 FAIL → 中止并升级为"人工仲裁"，把 FAIL 项与分歧点交用户定夺，不无限空转。

## 阶段四 · 人工审核提交

**角色**：提交者 / 待审协调员。
**目标**：把草稿与自查报告一并交付用户，按反馈迭代，直至确认后出 txt 并更新索引。

### 输入
- 阶段二草稿 md + 阶段三《自查报告》（全 PASS）。

### 提交物
- 草稿文件：`第六卷_乾坤/第{章号}_《章名》.md`
- 自查报告：随回复贴出（13 维 + 读者视角 + 逻辑矩阵 + 机械校验 全 PASS 结论）。
- 一句话连续性备忘（本章推进了哪些 FB / 角色状态变化 / 下章钩子）。

### 执行步骤与状态机
1. 将草稿 + 自查报告提交用户，标注"待审核"。
2. **等待用户反馈**（本阶段为终止态，不自主推进）。
3. 反馈分支：
   - **打回修改**：用户给具体意见 → 携带意见回阶段二/三修订（意见映射到对应自检条目），修订后重新提交。
   - **小修即过**：用户口头确认微调 → 改后直接重新提交（可免完整循环若改动极小，但仍需贴自查）。
   - **通过**：用户确认 → 执行"确认后"动作：
     a. 生成 `txt/第六卷_乾坤/第{章号}_《章名》.txt`（严格按 `写作规则.md` §三 txt 格式）；
     b. 更新治理索引：若本章新埋/回收 FB、新出场角色、新设定，回填 IDX_*（与大纲已登记项对齐，不新造号）；**并强制 `diff` 索引**（grep 正文出现的 FB 码/伏笔/角色 vs 索引登记章号，发现章号漂移即同步修正，如 FB-007 交付章与索引登记章须一致，防索引漂移）；
     c. 追加 daily memory 写作记录。
3.5 **反馈沉淀机制**（闭环优化）：用户打回的**共性模式问题**（非一次性笔误，如"战斗又写长""反派又降智""揭秘又直白喂"）→ 沉淀至 `写作规则.md` §四 或本项目 memory「写作反馈知识库」，下次 Stage1 必读 / Stage2 速查卡纳入；若属手法库缺失 → 扩展本附映射库。一次性笔误不沉淀。与阶段三 #7 **问题反补闸门**互补：#7 管"AI 自检/读者审查发现的新问题类型"，本步管"用户打回的共性模式"，二者都汇入 `docs/流程体系_小说创作.md` §2「问题→改进表」，形成写作对话内闭环 + 用户反馈外闭环的双重反补。
4. 一段（15 章）写完后，回看卷首说明五幕骨架做一致性抽检。

### 进入下一章的触发
- 当前章"通过并出 txt"后，用户指定下一章号 → 重新从阶段一启动（新章号解析 → 新必读集合）。

## 阶段五 · 审查对话（审查 + 修复，独立对话 · 第二道关）

**角色**：审查员 / 修复者（本对话，与写作对话分离）。
**目标**：作为**独立第二双眼睛**，对写作对话已做"自检 + 沉浸式读者审查 + 自修 + 反补"的 md 初稿再做一轮独立复核，专抓写作对话的**盲点**（如生成重复伪影、隐性 OOC、读者视角自查易放过的破墙），发现问题直接修复，输出《读者视角审查报告》。

> **双闭环衔接**：写作对话在其阶段三已完成"第一道关"内闭环（写作 → 自检 → 沉浸式读者审查 → 就地修改 → 问题反补 → 出"已自审稿"）；本阶段是**第二道关**——不重复基础自检，而是以更严格的独立视角深挖残留/盲点，并复核写作对话的"问题反补"是否到位。两者共用同一套预防闸门（`docs/流程体系_小说创作.md` §2「问题→改进表」），形成闭环。
> **直接修复权限**：店名/专名统一、年号/年龄校正、索引章号同步、生成重复/硬伤修复、R19 非例外现代词清除（经营理念现代词按 `IDX_PERSONA.md` §1 锁词表放行，不在此列；均属"既有内容修正"，非创作取向改写）。
> **创作取向建议**（章末钩子增强 / 修炼章节奏 / 反派密度）交写作对话执行，不擅自改写正文基调。
> **读者视角为唯一标尺**：机械校验 / 索引章号 / 大纲笔误属后台自检，不计入读者审查报告；仅当引发读者出戏才升格为审查项。

### 输入
- 写作对话产出的 md 初稿 + 磁盘索引/大纲（唯一事实源）。

### 执行步骤
1. **量化扫描**：字数、章末钩子命中率、破墙词（R19 现代词，经营理念词按 `IDX_PERSONA.md` §1 锁词表放行不计）、对话比（脚本/命令见 `docs/流程体系_小说创作.md` §7）。
2. **机械校验**：`python logic_check.py <章md>` → **C1–C22** 全过（C22 章内句级重复为新增闸门，防 117 章类生成伪影）。
3. **逐章读者视角深读**：爽点 / 节奏 / 人设 OOC / 破墙 / 可读矛盾（店名突变·年龄矛盾·死人复活·时间线乱·伏笔乱收）/ 章末钩子与伏笔满意度。
4. **索引交叉比对**：FB 码 / 伏笔 / 角色 / 设定 与正文一致，无悬空码、无章号漂移（如 FB-007 交付章与索引登记章一致）。
5. **处置**：硬伤 → 直接落盘修改（本对话权限）；创作取向 → 给写作对话执行。
6. **输出**：`审读_NNN-NNN读者视角.md`（纯读者视角；后台自检不计入报告）。

### 衔接触发逻辑（GATE-5）
- 验收标准（Definition of Done，见 `docs/流程体系_小说创作.md` §5）全部达成 → 标记"可出 txt"，交写作对话按"确认后"生成 txt + 回填索引。
- 任一硬伤未修 / 读者视角硬伤未消 → 不标记可出 txt。

### 反馈闭环（持续优化）
- 每轮审查发现 → 映射 `docs/流程体系_小说创作.md` §2「问题→改进表」；无对应闸门则新增一行（补"类"不是"个"）。
- 共性问题沉淀至 `写作规则.md` / 本体系；一次性笔误不沉淀。
