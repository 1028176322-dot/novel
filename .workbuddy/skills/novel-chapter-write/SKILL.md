# 章节正文四阶段写作流程（强制入口）

> **路径约定（可移植性）**：本 skill 已纳入小说项目仓库（位于 `<仓库根>/.workbuddy/skills/novel-chapter-write/`）。本文档及技能引用的所有文件路径，**均相对于小说项目仓库根目录**（即包含 `第一卷_道生/`、`写作规则.md`、`AI写作自动化流程.md` 的目录）。另一台电脑克隆仓库后，将仓库根设为 WorkBuddy workspace 即可直接使用，无需修改任何路径。
> 权威流程文档：`AI写作自动化流程.md`（位于仓库根，与本 skill 同步；若冲突以该文档为准）。
> 本 skill 是**唯一合规的章节正文撰写入口**。任何章节 prose 产出前，必须先跑完阶段一与阶段三。

## 铁律（不可绕过）
- 阶段一未 100% 加载 12 项必读文件 → **绝不写作**。
- 阶段三未贴逐条自查（14 维 + 读者视角审查 + 逻辑一致性防御矩阵 + 机械校验）→ **绝不提交**。
- 阶段四用户未确认 → **绝不生成 txt**。
- 红线 R1–R21、禁复活名单、FB 编号规则 → 硬约束，流程无权豁免。
- 审查对话分工：本 skill（写作对话）只生成初版 md；审查+修复由独立审查对话执行（见 `AI写作自动化流程.md` 阶段五 / `docs/流程体系_小说创作.md`），不在此纠正既有内容。
- 修订循环上限 N=3，超限升级人工仲裁，不无限空转。
- 阶段二动笔前**必须列《手法调度表》**（逐拍标剧情类型→叙述手法），未列不写作。
- **问题反补铁律**：阶段三 #7 发现的**新问题类型**（非一次性笔误）必须回写流程——可机检 → 提 `logic_check.py` C 码需求；预防类 → 加下方防复发清单（⑬⑭…）；流程缺漏 → 改 `AI写作自动化流程.md` / `写作规则.md`；红线例外 → 登 `IDX_PERSONA.md` 锁词；并沉淀至 `docs/流程体系_小说创作.md` §2「问题→改进表」（补"类"不是"个"）。一次性笔误不沉淀。

## 四阶段执行步骤

### 阶段一 · 写作前准备（加载器/守门员）
1. 解析用户给的章号 → 定位卷与大纲段文件。
2. 加载 12 项必读（见下"必读文件路径"），逐项输出《准备完成清单》✅/❌。
3. **GATE-1**：12 项全 ✅ 且 `AUTHORITY_INDEX.md` 中本卷大纲状态=current，才进阶段二；否则中止并报缺项。

### 阶段二 · 章节正文写作（作者态）
- **动笔前·认知边界预检（强制 · 全员分层）**：按 `IDX_COGNITION.md` 明确本章**所有出场角色**认知域——认知非主角专属，全员分层。主角先算年龄（公元−1370）定位 S0–S4 阶段取可知/不可知；其他角色查 §3 认知域（如无为子不知 FB-002、萧俊至死不知 FB-115 寒髓引毒、市井不知朝堂）。输出《认知边界备忘》（逐角色：POV/年龄阶段/可知/不可知/其他角色越界/空间锚点/可达边界），随稿提交自查对照（见流程文档阶段二·认知边界预检）。
- **动笔前·人设行为边界预检（强制 · 防 OOC/L4）**：按 `IDX_PERSONA.md` 明确本章**每个出场角色**性格—行为边界。主角取 §1 现代灵魂守则（现代思维仅内心/反差爽点、台词旁白零现代词 R19、不泄 FB-002、藏拙 R17、随年龄阶段性格弧；R19 例外：经营理念现代词仅限肖凡，见 `IDX_PERSONA.md` §1 锁词表）；其他角色取 §2 档案（性格标签/说话风格/行为决策/雷区，如无为子不涉权谋、萧俊不知 FB-115、苏墨凝不早于第082章）；反派取 §3 不降智有自身逻辑；新角色须带独特性格标签（§5）。输出《人设行为备忘》（逐角色：性格标签/声口/行为由性格+处境驱动？/触雷区？/主角专项），随稿提交 Stage3 L4 逐条对照（见流程文档阶段二·人设行为边界预检）。
0. **手法调度（强制先列后写）**：动笔前先列《手法调度表》——逐拍标注剧情类型→叙述手法（见流程文档"阶段二·附 叙述手法调度规范"映射库）、角色声音、单章节奏七步。未列表不写作。
- **动笔前·防复发自检清单（强制，对应 C8–C22 精神）**：列完手法调度表后、动笔前，逐条过——① 章名是否同卷重名（防 C8）？② 是否复制粘贴前文段落（防 C9）？③ 若写开火是否已有「藏/暗/武籍」缓冲且不违火器必藏(R17)（防 C10）？④ 有无「第X章那回/幕X的/他将图收起」等编辑残留（防 C11）？⑤ 年龄/年号是否按 1370=永熙三年=0岁 推算一致（防 C12）？⑥ 「前世」是否过多、可否改动作呈现（防 C13）？⑦ 新角色名是否与既有角色（如钱万山）易混（防 C14）？⑧ 引号内台词是否出现现代实物名词/网络梗（防 C15，R19 仅限内心独白可现代词；经营理念词按 `IDX_PERSONA.md` §1 锁词表放行不告警）？⑨ 纯汉字字数是否 2500–3000（防 C16）？⑩ 章名是否 ≥4 字（防 C17，第7章起）？⑪ 专名 canonical 一致性：商号/人名/地名是否取大纲/索引 canonical 名、全卷无旧名回潮（防 81–100 店名断裂）？⑫ 章内是否出现相邻重复句（防 C22 生成伪影，如 117 章类）？
1. 严格贴 v2 大纲本章"情节节拍分解"逐拍展开，且每拍用调度表指定的手法写。
2. 单章 **不含标点的纯汉字 2500–3000**；每章 ≥1 小爽点 + 章末钩子；单章节奏按七步配速。
3. 红线/禁复活/FB 严守；新角色按索引占位名先留白（如"新国师"）；角色声音按差异化表。
4. **写完扫描（防备注漏入正文）**：落笔后通扫全章，凡 `（节拍…）`/`（R\d+：…）`/`（章末钩子…）`/`（悬念钩子…）` 等作者写作备注/手法标注/红线标记/钩子说明括号行，一律删除——只存于大纲与《手法调度表》，**绝不进正文**。
5. **GATE-2**：非空 + 字数达标 + 有钩子 + 无写作备注混入，才进阶段三。

### 阶段三 · 写作自检 + 沉浸式读者审查（内闭环·审查员态）
0. **机械校验（强制先于一切）**：运行 `python .workbuddy/skills/novel-chapter-write/logic_check.py <本章md>`（脚本会自动按仓库根定位治理目录 `docs/governance`；`<本章md>` 用相对仓库根的路径，如 `第一卷_道生/第003章_探寻.md`），输出 **C1–C22**（含 C22 章内句级重复、C23 专名回归）。其中 **C8–C22 为读者审读报告 P0/P1 修复后追加的防复发检查**：C8 章节命名唯一性、C9 跨章段落雷同(≥5行)、C10 火器必藏设定、C11 编辑残留/第四面墙、C12 纪年年龄自洽、C13 前世口头禅频次、C14 角色命名冲突、C15 台词现代词(仅引号内·内心独白豁免·经营理念词按 `IDX_PERSONA.md` §1 锁词表放行)、C16 字数达标(纯汉字2500–3000)、C17 章名≥4字(第7章起)。**C1–C6、C8、C11、C18、C19、C20 任一 FAIL → 直接回阶段二修订，不进入后续 AI 核查**（C8/C11/C18/C19/C20 为硬 FAIL）；C7、C9、C10、C12、C13、C14、C15、C16、C17、C21、C22 为 WARN 级，须在读者视角审查 / 逻辑矩阵 L9 / 字数·章名核查 逐条显式回应（未回应视为未自查）。
1. 跑 13 维规则核验（含逻辑项须查"角色认知边界·全员分层"：所有角色只写其认知域可知信息，主角按年龄阶段 S0–S4 取可知/不可知，不写角色认知域外信息、不替读者让角色提前确知未回收伏笔答案，受限视角不描写自身不可见物外观仅写触觉/听觉——对照 `IDX_COGNITION.md`；**含 L4 角色性格与行为逻辑一致性**：每个出场角色言行命中其性格标签、说话风格匹配声口、行为由性格+处境驱动非作者强排、主角现代灵魂仅内心/台词旁白零现代词(R19；R19 例外：经营理念现代词仅限肖凡，见 `IDX_PERSONA.md` §1 锁词表)/不泄 FB-002/随年龄阶段性格弧、反派不降智有自身逻辑(§3)、新角色有独特性格标签非工具人——对照 `IDX_PERSONA.md` §2/§1/§4；**含 #13 句级衔接/突兀冗余**：逐句（尤其独立成段的导语句）过三问（承接/一致/冗余），提交前段首句扫描，独立成段单句必走三问——**冗余问须双向**（前向查"后文是否又演一遍"、反向查"前文是否已用具体场景演出过同等内容"，**show 之后再 tell 复述即 FAIL**）——见流程文档阶段三 #13）。
2. 跑**读者视角审查（全维度 24 项，见下方「读者视角审查清单」）**；以**首次读者身份逐章追更模拟**，凡 FAIL 项**立即就地修改**正文后重跑至 PASS（内闭环"审查→修复"，不依赖审查对话）。
3. 跑**逻辑一致性防御矩阵**（L1–L9，见流程文档阶段三·附；L9 空间须逐条回应 C7 的 WARN）。
4. 在回复中**贴出逐条自查结果**（13 维 + 读者视角 + 逻辑矩阵 + 机械校验，强制，不贴视为未自查）。
4.5 **问题反补闸门（闭环优化·强制）**：本次审查发现的**新问题类型**（非一次性笔误）必须反补——可机检 → 提 `logic_check.py` C 码需求；预防类 → 加下方防复发清单（⑬⑭…）；流程缺漏 → 改 `AI写作自动化流程.md` / `写作规则.md`；红线例外 → 登 `IDX_PERSONA.md` 锁词；并同步 `docs/流程体系_小说创作.md` §2「问题→改进表」新增一行（补"类"不是"个"）。一次性笔误不沉淀。
5. **GATE-3**：14 维 + 读者视角审查(全维度 24 项) + 逻辑矩阵(L1–L9) 全 PASS 且 logic_check.py(**C1–C6、C8、C11、C18、C19、C20 无 FAIL**；C7/C9/C10/C12/C13/C14/C15/C16/C17/C21/C22 的 WARN 已在读者视角审查 / 逻辑矩阵 L9 / 字数·章名核查 逐条显式回应) **且问题反补闸门(4.5)已执行** → 进阶段四；否则带 FAIL 项回阶段二修订。

### 阶段四 · 人工审核提交（待审协调员）
1. 交草稿 md + 自查报告，标注"待审核"，等用户反馈。
2. 打回 → 按意见回阶段二/三；通过 → 生成 `txt/卷名/...`（仅确认后）+ 回填索引 + 写 daily memory。
3. 下一章：用户指定新章号 → 从阶段一重启。

## 读者视角审查清单（阶段三强制块 · 全维度 24 项）

**全维度无死角原则**：读者能感知的一切点都纳入审查——爽感、节奏、人物、世界、文字、伏笔 6 大类共 24 项（权威定义见 `docs/流程体系_小说创作.md` 附录 A）。逐行以首次读者身份审，判定"应见 / 禁见"：

- **A 爽感（A1–A4）**：轻视→打脸爽点可感、情绪有起伏、悬念信息差（读者自拼答案）、现代vs古代反差爽感（FB-002 不泄）。
- **B 节奏（B1–B5）**：章内无灌水、章末有截断钩子、战斗/对话/修炼交替不疲劳、设定自然露出（不 show后tell）、战斗写结果不写过程。
- **C 人物（C1–C5）**：人设不 OOC、动机合理、对话像真人（无说明性对话/现代词入台词）、反派不降智/配角有记忆点、成长用事件呈现。
- **D 世界（D1–D4）**：时代氛围统一（无破墙）、场景五感落地有锚点、设定前后自洽、称谓关系清晰不混。
- **E 文字（E1–E3）**：通顺无病句/无孤立突兀句、时空跨度自洽（无瞬移无手段）、因果逻辑不断（战力/资源有来源）。
- **F 伏笔（F1–F3）**：伏笔埋巧收稳、已死/已退角色去向干净、长生机制（FB-005）逐步深化不剧透。

> 任一项"禁见"命中即 FAIL，**立即就地修改**对应正文后重跑至 PASS（内闭环）。

## 手法适配核查（阶段三强制项）
逐拍对照"剧情类型→叙述手法映射库"：战斗用结果速写（不招式注水）、揭秘用渐进揭示（不直白喂）、权谋用信息差（不亮底牌）、章末钩用截断悬念（不先说答案）、角色声音按差异化表。任一节拍手法错配 → FAIL 回阶段二。
