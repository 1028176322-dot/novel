# 章节正文四阶段写作流程（强制入口）

> **路径约定（可移植性）**：本 skill 已纳入小说项目仓库（位于 `<仓库根>/.workbuddy/skills/novel-chapter-write/`）。本文档及技能引用的所有文件路径，**均相对于小说项目仓库根目录**（即包含 `第一卷_道生/`、`写作规则.md`、`AI写作自动化流程.md` 的目录）。另一台电脑克隆仓库后，将仓库根设为 WorkBuddy workspace 即可直接使用，无需修改任何路径。
> 权威流程文档：`AI写作自动化流程.md`（位于仓库根，与本 skill 同步；若冲突以该文档为准）。
> 本 skill 是**唯一合规的章节正文撰写入口**。任何章节 prose 产出前，必须先跑完阶段一与阶段三。

## 铁律（不可绕过）
- 阶段一未 100% 加载 12 项必读文件 → **绝不写作**。
- 阶段三未贴逐条自查（13 维 + 读者视角审查 + 逻辑一致性防御矩阵 + 机械校验）→ **绝不提交**。
- 阶段四用户未确认 → **绝不生成 txt**。
- 红线 R1–R21、禁复活名单、FB 编号规则 → 硬约束，流程无权豁免。
- 修订循环上限 N=3，超限升级人工仲裁，不无限空转。
- 阶段二动笔前**必须列《手法调度表》**（逐拍标剧情类型→叙述手法），未列不写作。

## 四阶段执行步骤

### 阶段一 · 写作前准备（加载器/守门员）
1. 解析用户给的章号 → 定位卷与大纲段文件。
2. 加载 12 项必读（见下"必读文件路径"），逐项输出《准备完成清单》✅/❌。
3. **GATE-1**：12 项全 ✅ 且 `AUTHORITY_INDEX.md` 中本卷大纲状态=current，才进阶段二；否则中止并报缺项。

### 阶段二 · 章节正文写作（作者态）
- **动笔前·认知边界预检（强制 · 全员分层）**：按 `IDX_COGNITION.md` 明确本章**所有出场角色**认知域——认知非主角专属，全员分层。主角先算年龄（公元−1370）定位 S0–S4 阶段取可知/不可知；其他角色查 §3 认知域（如无为子不知 FB-002、萧俊至死不知 FB-115 寒髓引毒、市井不知朝堂）。输出《认知边界备忘》（逐角色：POV/年龄阶段/可知/不可知/其他角色越界/空间锚点/可达边界），随稿提交自查对照（见流程文档阶段二·认知边界预检）。
- **动笔前·人设行为边界预检（强制 · 防 OOC/L4）**：按 `IDX_PERSONA.md` 明确本章**每个出场角色**性格—行为边界。主角取 §1 现代灵魂守则（现代思维仅内心/反差爽点、台词旁白零现代词 R19、不泄 FB-002、藏拙 R17、随年龄阶段性格弧）；其他角色取 §2 档案（性格标签/说话风格/行为决策/雷区，如无为子不涉权谋、萧俊不知 FB-115、苏墨凝不早于第082章）；反派取 §3 不降智有自身逻辑；新角色须带独特性格标签（§5）。输出《人设行为备忘》（逐角色：性格标签/声口/行为由性格+处境驱动？/触雷区？/主角专项），随稿提交 Stage3 L4 逐条对照（见流程文档阶段二·人设行为边界预检）。
0. **手法调度（强制先列后写）**：动笔前先列《手法调度表》——逐拍标注剧情类型→叙述手法（见流程文档"阶段二·附 叙述手法调度规范"映射库）、角色声音、单章节奏七步。未列表不写作。
1. 严格贴 v2 大纲本章"情节节拍分解"逐拍展开，且每拍用调度表指定的手法写。
2. 单章 **不含标点的纯汉字 2500–3000**；每章 ≥1 小爽点 + 章末钩子；单章节奏按七步配速。
3. 红线/禁复活/FB 严守；新角色按索引占位名先留白（如"新国师"）；角色声音按差异化表。
4. **写完扫描（防备注漏入正文）**：落笔后通扫全章，凡 `（节拍…）`/`（R\d+：…）`/`（章末钩子…）`/`（悬念钩子…）` 等作者写作备注/手法标注/红线标记/钩子说明括号行，一律删除——只存于大纲与《手法调度表》，**绝不进正文**。
5. **GATE-2**：非空 + 字数达标 + 有钩子 + 无写作备注混入，才进阶段三。

### 阶段三 · 写作自检（审查员态）
0. **机械校验（强制先于一切）**：运行 `python .workbuddy/skills/novel-chapter-write/logic_check.py <本章md>`（脚本会自动按仓库根定位治理目录 `docs/governance`；`<本章md>` 用相对仓库根的路径，如 `第一卷_道生/第003章_探寻.md`），输出 C1–C7。C1–C6 任一 FAIL → 直接回阶段二修订，不进入后续 AI 核查；C7 为空间 WARN 级（锚点缺失/互斥地点冲突），须在 L9 逐条显式回应。
1. 跑 13 维规则核验（含逻辑项须查"角色认知边界·全员分层"：所有角色只写其认知域可知信息，主角按年龄阶段 S0–S4 取可知/不可知，不写角色认知域外信息、不替读者让角色提前确知未回收伏笔答案，受限视角不描写自身不可见物外观仅写触觉/听觉——对照 `IDX_COGNITION.md`；**含 L4 角色性格与行为逻辑一致性**：每个出场角色言行命中其性格标签、说话风格匹配声口、行为由性格+处境驱动非作者强排、主角现代灵魂仅内心/台词旁白零现代词(R19)/不泄 FB-002/随年龄阶段性格弧、反派不降智有自身逻辑(§3)、新角色有独特性格标签非工具人——对照 `IDX_PERSONA.md` §2/§1/§4；**含 #13 句级衔接/突兀冗余**：逐句（尤其独立成段的导语句）过三问（承接/一致/冗余），提交前段首句扫描，独立成段单句必走三问——**冗余问须双向**（前向查"后文是否又演一遍"、反向查"前文是否已用具体场景演出过同等内容"，**show 之后再 tell 复述即 FAIL**）——见流程文档阶段三 #13）。
2. 跑**读者视角审查**（应见/禁见双列 12 项，含"正文纯净度"与"衔接连贯（孤立突兀句）"项）。
3. 跑**逻辑一致性防御矩阵**（L1–L9，见流程文档阶段三·附；L9 空间须逐条回应 C7 的 WARN）。
4. 在回复中**贴出逐条自查结果**（13 维 + 读者视角 + 逻辑矩阵 + 机械校验，强制，不贴视为未自查）。
5. **GATE-3**：13 维 + 读者视角审查 + 逻辑矩阵(L1–L9) 全 PASS 且 logic_check.py(C1–C6 无 FAIL；C7 空间 WARN 已在 L9 逐条显式回应) → 进阶段四；否则带 FAIL 项回阶段二修订。

### 阶段四 · 人工审核提交（待审协调员）
1. 交草稿 md + 自查报告，标注"待审核"，等用户反馈。
2. 打回 → 按意见回阶段二/三；通过 → 生成 `txt/卷名/...`（仅确认后）+ 回填索引 + 写 daily memory。
3. 下一章：用户指定新章号 → 从阶段一重启。

## 读者视角审查清单（阶段三强制块）
逐行以首次读者身份审，判定"应见 / 禁见"：
- **应见**：设定自然露出、线索自己拼答案、现代vs古代反差爽感、长生逐步深化（FB-005）、权谋信息差悬念、轻视→打脸爽点、伏笔回味、画面与情绪、战果与爽感、沉浸感统一。
- **禁见**：作者跳出来解说/教科书堆砌、直白喂结论、泄底穿越者身份（FB-002）、长生剧透终局、底牌同时亮明、爽点泄气/反派降智、硬塞未回收伏笔、流水账、招式注水、视角漂移/出戏、**孤立突兀句（单句抽离成段无承接、与紧邻段落状态/时间冲突、或与后文内容冗余）**、**show 之后又 tell 复述（对话/动作/感官已演出的性格/设定/信息，再用概括句贴标签复述一遍，属隐性作者视角）**。

## 手法适配核查（阶段三强制项）
逐拍对照"剧情类型→叙述手法映射库"：战斗用结果速写（不招式注水）、揭秘用渐进揭示（不直白喂）、权谋用信息差（不亮底牌）、章末钩用截断悬念（不先说答案）、角色声音按差异化表。任一节拍手法错配 → FAIL 回阶段二。
